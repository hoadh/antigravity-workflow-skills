## Phase 1: Codebase Scouting

1. Run `ls -la` to identify actual project directories
2. Search the codebase across 2-4 key directories in parallel
3. Merge results into context summary

## Phase 2: Documentation Creation

Create initial documentation using the gathered file list:
- `docs/project-overview-pdr.md`: Project overview and PDR
- `docs/codebase-summary.md`: Codebase summary
- `docs/code-standards.md`: Codebase structure and code standards
- `docs/system-architecture.md`: System architecture
- Update `README.md` with initial documentation (keep under 300 lines)

Use `docs/` directory as source of truth.

**Do not** start implementing code.
